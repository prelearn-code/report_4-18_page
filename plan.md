去中心化安全存储系统 (D-Storage) 可视化演示方案

本文档旨在规划一个纯前端的交互式可视化沙盒（Simulator / Playground）。它的目的不是作为生产环境可用的网盘客户端，而是作为辅助论文阅读、项目答辩或教学演示的直观工具，用以生动展现混合审计、加密去重、HVT 更新等复杂密码学流程。

1. 核心设计理念

“将密码学黑盒变成透明的流水线”

无真实密码学计算：放弃在前端使用 Web Worker 进行高耗时的模幂或配对运算，改用 setTimeout 配合 CSS 动画模拟“计算中”的耗时感。

状态可视化取代数据持久化：不再使用 IndexedDB 保存大文件或真实的密钥，而是用状态机控制 UI 面板的切换和高亮。

全局控制台 (System Terminal)：在页面底部设计一个全局日志输出窗口。所有的动作背后（例如客户端生成认证器、合约计算双线性映射），都以人类可读的“伪代码”或“日志描述”形式打印出来。

2. 四大交互演示模块设计

沙盒将通过顶部 Tab 切换进入不同的演示模块，每个模块针对论文中的一个核心创新点。

2.1 模块一：加密去重与分块上传 (Deduplication)

展示目标：让观众看懂“私密块”与“公开块”是如何在不泄露隐私的前提下实现去重的。

视觉元素：

左侧：客户端实体（生成数据块的图标）。

右侧：云服务 CSP 实体。

中间：一组可视化的卡片，代表拆分后的数据块 (Blocks)。

交互流程 (步进动画)：

点击【演示去重】：出现 3 个原始块卡片。

本地计算：卡片样式变化，打上哈希标签（如 tg_1），模拟加密过程。

云端碰撞：向 CSP 发出检查。Block 2 突然变成红色半透明（标记为 Duplicate），Block 1/3 变成亮绿色发光（标记为 Unique）。

传输：只有亮绿色的卡片飞向 CSP 实体完成存储。

2.2 模块二：HVT 树动态更新 (HVT Topology)

展示目标：直观反映基于哈希的版本树（Hash Version Tree）在面临局部更新时，Hash 和 Weight（块数） 如何同步变化。

视觉元素：

采用简化的层级结构（Flexbox 嵌套）绘制一棵三层的二叉树。

每个节点是一个信息面板，展示：Hash 变量名、Weight 值、Index 序号。

交互流程 (状态切换)：

提供四个状态按钮：【初始状态】、【修改 (Modify m1)】、【删除 (Delete m1)】、【插入 (Insert m* after m1)】。

点击修改：受影响节点的边框变蓝，哈希值加上星号（$h_1 \rightarrow h_1^*$），但所有节点的 Weight 保持不变。

点击删除：目标叶子节点消失，兄弟节点上移。沿途父节点的边框变红，展示 Weight 值实时 -1。

点击插入：目标节点下方分裂出新的子节点树。沿途父节点的边框变绿，展示 Weight 值实时 +1。

2.3 模块三：智能合约混合审计 (Hybrid Audit)

展示目标：展示去信任化环境中，智能合约如何通过 Challenge-Proof 机制自动抽查云端数据。

视觉元素：

呈三角形排布的三个大图标：用户、智能合约、CSP。

图标之间通过流动光带（CSS 动画）展示数据流向。

交互流程 (步进动画)：

用户向智能合约连线，点亮合约。

合约广播 Challenge 变量，连线指向 CSP。

CSP 图标开始闪烁（模拟聚合证明计算），生成 Proof 并传回合约。

合约执行验证，屏幕中心弹出绿色的“验证通过”标识。

2.4 模块四：数据所有权交易 (Data Market) - (预留扩展)

展示目标：说明标签/认证器如何与所有者解绑，实现秒级所有权交割。

视觉元素：买家 Alice、卖家 Bob、CSP 记录表。

核心演示点：点击交易后，重点展示 CSP 仅需更改表中的 Owner ID（而不需要重算树根或标签），最后卖家通过信封（私信）将解密密钥 sk 传递给买家即可。

3. 技术栈建议

框架：React (单文件组件足以承载)。

样式：Tailwind CSS（特别利用其 transition 和 animate 工具类来实现卡片变色、发光等效果）。

图标：lucide-react（用于生动表达服务器、数据库、文件、加密锁等概念）。

通过这样一个纯视觉和交互驱动的沙盒前端，任何阅读论文的人都可以“边玩边学”，极大地降低了理解加密协议中抽象公式的门槛。
